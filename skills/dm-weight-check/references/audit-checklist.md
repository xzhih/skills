# 减重检查清单

这个文件不是主说明。只有在 `dm-weight-check/SKILL.md` 已经触发，并且需要更细地检查方案是否变重时再读。

重点检查这些地方：

```text
核心
运行时协调层
注册和发现
记忆
自动化流程
调试
安装和更新
外部能力或插件
用户界面
```

逐项问：

```text
我们是不是因为一个功能看起来重要，就把它放进核心了？
我们是不是把用户看到的界面状态，当成了系统执行状态？
我们是不是因为以后可能有用，提前留下了太多东西？
我们是不是把授权、加载、放进上下文、允许执行混成了一件事？
我们是不是把注册表做成了市场？
我们是不是把调试做成了默认全量追踪平台？
我们是不是把记忆做成了资料库或流程仓库？
我们是不是让自动化或插件自己在后台运行？
```

变重的迹象：

```text
核心拥有越来越多名词
一个文档里不断出现 Manager 和 Gateway
流程成了默认容器
记忆开始保存完整步骤
调试默认保留原始请求和响应
安装功能开始像市场
插件想自主后台运行
测试离不开真实平台、真实模型或真实外部服务
```
