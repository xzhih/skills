# 注意事项

这个文件不是主说明。只有在讨论开始跑偏、过早进入实现、或者系统中心开始变重时再读。

常见跑偏方式：

```text
架构还没清楚就开始讨论界面
过早固定接口、协议、跨语言绑定细节
把“已安装”当成“已经进入上下文”
把记忆做成流程仓库
把自动化流程做成默认系统容器
把注册表做成市场
把调试做成默认全量追踪平台
把所有未来功能都往核心里放
```

这些词不禁止使用，但要小心，因为它们经常掩盖复杂度变重：

```text
Manager
Gateway
Trace
Workflow
Memory system
Plugin platform
Marketplace
Developer console
Role binding
```
